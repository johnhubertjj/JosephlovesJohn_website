Repository Reading Plan

A structured reading order for understanding the JosephlovesJohn codebase end to end, with the smallest number of files that still explain how the site works in production.

Docs Map

  • docs/architecture/ - repo-level architecture and system diagrams.
  • docs/shop/ - shop, cart, checkout, order, and download flow notes.
  • docs/deployment/ - Render, Cloudflare R2, storage, and versioning notes.
  • docs/performance/ - performance report workflow and benchmarking notes.
  • docs/launch/ - launch and prelaunch checklists.
  • docs/seo/ - Search Console and SEO operations notes.
  • docs/reading-plans/ - reading plans for the repo and focused features.
  • docs/local-notes/ - local/private notes that are useful on this machine but not general project docs.

Recent Mastering Changes to Notice

  • mastering/forms.py separates artist and mix engineer intake questions while keeping both submissions compatible with one Google Sheet.
  • mastering/views.py can email intake submissions and optionally post a structured payload to MASTERING_INTAKE_GOOGLE_SHEETS_WEBHOOK_URL.
  • templates/mastering/home.html contains page-specific Solid State styling for the hero, examples, pricing, services, contact, and FAQ sections.
  • templates/mastering/intake.html handles role-specific field visibility and custom checkbox/radio styling.

Recent Security Changes to Notice

  • josephlovesjohn_site/csp.py now sends a Content Security Policy allowing only the expected first-party, Plausible, Kit/ConvertKit, Stripe, and configured asset origins.
  • shop/views.py applies no-cache protections to account, authentication, checkout, cart, success, and download views so browser history does not revive stale private pages after logout.
  • templates/shop/base.html includes a back-forward-cache guard for signed-in shop pages.
  • shop/forms.py and templates/shop/register.html now use a registration honeypot, and registration rate limiting is configured through REGISTER_RATE_LIMIT_ATTEMPTS and REGISTER_RATE_LIMIT_WINDOW.
  • shop/models.py no longer stores customer profile full names; account creation only needs username, email, and password.
  • render.yaml documents the production-facing CSP and registration rate limit environment variables, though Render may still be managed manually.
